# AGB CRM — UX/UI Critique
**Reviewer:** OVL-AGB-Claude · **Date:** 2026-05-26 · **Scope:** 33 routes, all components, every state

Method: code read of every page + component, route probing via curl against the
running dev server, contrast/typography audit against the design tokens. Visual
screenshot capture attempted but blocked by disk-full state (`/` at 99%);
critique therefore relies on rendered HTML + code.

## TL;DR

The system has solid bones — clear design tokens (OKLCH, dark-mode-ready), a
consistent server-actions-with-form pattern, real empty / DB-missing /
loading states, and a coherent component library. **The weak spots are
mobile, feedback-after-action, and small-detail polish that compounds across
33 routes.** None are functional blockers; all are addressable in the same
day they ship.

Severity legend:
- **P0** breaks a category of users (mobile, AT, Safari)
- **P1** every user feels it as a paper cut
- **P2** missing power-user / nice-to-have features
- **P3** taste

## B. Per-surface findings

### /login

- **B1 [P1]** Bare layout — centered card, no logo, no product tagline, no fine print. For an internal tool this is fine, but a 32×32 brand glyph + a one-line explainer ("Internal — Founders only") would set expectations for first-time users.
- **B2 [P1]** No "Resend" CTA on the success state. If the magic-link email is delayed/lost, the user has to refresh the whole page.
- **B3 [P2]** No rate-limit messaging. If Supabase rate-limits, the `authError.message` is shown raw — fine but unstyled vs. design tokens.
- **B4 [P3]** `Check your inbox at <email>` truncates at the box edge with no `break-all`.

### / (This-Week)

- **B5 [P1]** The 3 stat cards use color-tinted ring borders (good), but the icons inside `CardTitle` are wrapped in a flex but no aria-label. AT reads the icon as decorative noise.
- **B6 [P1]** Heatmap renders without a legend. Users see green squares of varying intensity but no "less ← → more" key. Also no axis labels.
- **B7 [P1]** "Stale friends" list uses `daysSince === null ? "never" : "Xd ago"`. The "never" label is ambiguous (never touched? never met?). Prefer "no touches logged."
- **B8 [P2]** Three full-width sections (Due, Blocked, Stale) cause a lot of vertical scrolling on desktop. Consider tabs for the lower-priority groups.
- **B9 [P3]** Numeric values in stat cards aren't `tabular-nums` — digit width jitters when the count changes.

### /contacts

- **B10 [P1]** Toolbar mixes Plus + 30-sec buttons in the TopBar action area; the in-empty-state CTA *only* shows Plus. Inconsistent — empty state should also surface 30-sec as the faster onboarding path.
- **B11 [P1]** VenturePillBar renders even with zero tags (it returns null when empty, OK), but with one tag it shows just `[All] [caney]` which looks broken vs. a 4-tag set.
- **B12 [P2]** No bulk selection (per spec — Phase 6+), but the row hover state offers no affordance suggesting clickability outside of the name link.
- **B13 [P2]** "Last touch" sorting works, but a "stale" column (red for >60d on friends) would surface intent faster than a relative date.
- **B14 [P3]** `Active · Archived` toggle is a text link pair, not a Tabs component. Inconsistent with shadcn norms.

### /contacts/[id]

- **B15 [P1]** The TopBar packs three actions (ReintroButton, Edit, Archive form) without separators — visually crowded.
- **B16 [P1]** Voice recorder is below the manual touch form with a thin divider; no visual signal that they're *alternatives* for the same outcome. Tabs ("Type" / "Speak") would make the choice clear.
- **B17 [P1]** When `archive` succeeds, the page silently reloads. No confirmation. Same for unarchive.
- **B18 [P2]** ReintroButton draft opens *inline below the button*, pushing the rest of the page down. Should be a Sheet/Dialog so the page state doesn't shift.
- **B19 [P2]** Warm-path "via" tooltip text uses smart quotes (`&ldquo; &rdquo;`) — looks nice but inconsistent with the rest of the app (no other surface uses smart quotes).
- **B20 [P3]** ReciprocityCard heuristic disclosure is good, but the bar splits left=green right=amber, which can read as "green is good" — but high "you" share is actually the "they-owe" case. Could be misread.

### /contacts/new + /[id]/edit

- **B21 [P1]** Channels list always shows at least one row with kind=email and empty value. If user wants no channels, they have to leave the empty row and the server filters it out — but the UX implies "you must enter at least one."
- **B22 [P1]** Tags pill bar uses the same Badge variant treatment as Channels in the contact form — no visual hierarchy between *selecting* a venture vs. *adding* a row.
- **B23 [P2]** No "Save and add another" — adding 5 contacts in a row requires 5 round-trips.
- **B24 [P3]** No required-field markers (asterisk / "required" label). Only Name is required; users have to discover this by submitting.

### /contacts/quick

- **B25 [P0]** Safari MediaRecorder doesn't support `audio/webm`. The recorder will throw on Safari/iOS. Need `audio/mp4` fallback.
- **B26 [P1]** No replay-before-submit. User can't verify the recording before it gets transcribed.
- **B27 [P1]** No max-duration cap. "30-sec contact" implies 30s ceiling; nothing stops you from rambling for 5 minutes.

### /projects + /projects/[id] + /projects/new + /[id]/edit

- **B28 [P1]** Project status `lost` uses the same red Badge variant as a critical health red. Conflates "this didn't work out" with "this is on fire." Use `secondary` / neutral grey for `lost`.
- **B29 [P1]** Project detail page's "Spawn N milestones" button (from meeting minutes) appears on the meeting detail, not the project detail — discoverability inverted from what users expect.
- **B30 [P1]** When status=waiting, the project form expands with `waitingOn` + `expectedUnblockDate`. The expand uses no transition; jarring jump.
- **B31 [P2]** No "duplicate project" or "use as template" affordance — common workflow for repeat client patterns.
- **B32 [P3]** Project list shows "Open" column header for milestone count — ambiguous (open issues? open status?). Rename "Open milestones."
- **B33 [P3]** Milestone status badges (`pending`, `done`, `blocked`) use small badges; combined with the title text + relative date + blocker line it gets visually noisy. Consider a single status pip + tooltip.

### /meetings + /meetings/[id]

- **B34 [P1]** `new Date(m.scheduledAt).toLocaleString()` renders different formats in different locales — could be "5/26/2026, 2:00:00 PM" or "26/05/2026 14:00:00". Lock to a short format ("Tue, May 26 · 2:00 PM") via `Intl.DateTimeFormat`.
- **B35 [P1]** Action items detected in minutes display as `☐` glyphs but live milestones use Checkbox primitives. Two visual languages for "unchecked."
- **B36 [P2]** No meeting recurrence / template ("weekly 1:1 with Marta").
- **B37 [P2]** Meeting form attendee picker is a chip toggle — fine for 5–10 contacts, breaks down at 100+. Needs a search-filter.
- **B38 [P3]** Linked project select shows only project title; doesn't show whether the project is `active|done|lost`. Easy to attach a meeting to a lost project by mistake.

### /pipeline

- **B39 [P0]** Kanban card prev/next buttons appear on hover only (`opacity-0 group-hover:opacity-100`). Touch devices have no hover — controls are inaccessible. Use `group-focus-within` + always visible on touch, or always show at reduced opacity.
- **B40 [P1]** Stage columns are 288px (`w-72`) and cards don't wrap titles. Long project names truncate without a tooltip showing the full title.
- **B41 [P1]** Empty columns show "empty" — accurate but cheerless. "Drag a project here" or "no projects on this stage" would be more useful.
- **B42 [P2]** Health dot uses the same colors as the row Health badge but with no legend on the page.
- **B43 [P3]** Template pill bar uses `rounded-full border` chips, but the venture pill bar elsewhere uses rounded-full + filled-when-active. Inconsistent active styling.

### /network

- **B44 [P1]** Intro tree uses left-border dashed lines for depth, which looks fine ≤3 levels deep but breaks visually past that.
- **B45 [P1]** "No contacts to show under this lens" is correct but doesn't suggest the obvious next step (switch to All lens, or add intro chains via the contact form).
- **B46 [P1]** The trailing `<p>` tip references "forthcoming" — paper cut acknowledging incomplete UX (contact edit *does* support intro chain via text; but the parent-id picker doesn't exist yet).
- **B47 [P3]** WarmPath card lives on contact detail, not on /network. Discoverability inverted — users looking for warm paths look at /network first.

### /tags

- **B48 [P1]** Custom tag color uses native `<input type="color">` — different UX per browser; mobile is especially janky. Use a swatch picker.
- **B49 [P1]** No tag rename UX — only create/delete. Users will want to fix typos.
- **B50 [P2]** `personal-only` / `ai-ok` semantic tags are described in HANDOFF but invisible in the UI. The tag dictionary should explain that these tags drive silence rules.

### /profile

- **B51 [P1]** Email field is disabled (correct — Supabase owns it) but no helper text saying "managed by Supabase Auth."
- **B52 [P1]** Timezone uses a `datalist` — works but offers no validation. Users can type "Mars/Phobos" and submit.
- **B53 [P3]** No avatar upload (per spec — phase 6 polish). At least show a colored placeholder larger than the 8×8 dropdown trigger.

### /settings

- **B54 [P3]** Two links to sub-pages. This page exists to anchor `/settings` as a destination from the user menu, but it's almost content-free. Either fold settings into Profile and Tags directly, or add real settings (notification preferences, email digest opt-in, etc.).

## C. Component-level findings

### Sidebar / TopBar / UserMenu

- **C1 [P0]** No mobile nav. (A1 restated.)
- **C2 [P1]** Sidebar logo is `AGB` glyph inside a rounded square. Fine. But hover state doesn't change at all — no affordance that "AGB CRM" is clickable (it isn't, but the cursor doesn't communicate that either).
- **C3 [P1]** Nav item active state uses `bg-[var(--accent)]` — works in light mode, but in dark mode the accent is close to the card background and the active state is nearly invisible. Bump active state to `bg-[var(--accent)]/80` + thicker font-weight.
- **C4 [P2]** UserMenu initials fall back to `"F"` if displayName is empty — that's just letter F, not "Founder." Should be `"…"` or the first character of the email.
- **C5 [P3]** "Phase 0 · v0.1.0" sidebar footer is dead text. Either tie to real version (`process.env.npm_package_version`) or remove.

### Button / Input / Card / Badge / Select / Dialog / Dropdown

- **C6 [P1]** Input focus ring uses `focus-visible:ring-[var(--ring)]` (good) but the `ring-offset` color is `var(--background)` which is fine in light mode but blends with the field background in dark mode — ring appears thin.
- **C7 [P1]** Button `loading` state isn't a variant — every form re-implements `{pending ? "Saving…" : "Save"}`. Add a Spinner / `loading` prop to Button.
- **C8 [P1]** Card has `shadow-sm` always — on dark mode this is invisible. Use `shadow-sm` for light, `ring-1 ring-white/5` for dark.
- **C9 [P2]** No Toast usage in spite of `<Toaster />` being mounted.
- **C10 [P3]** Select trigger is `h-9` while Button default is `h-9` (good) but Input + Textarea use `h-9` and `min-h-[72px]` — consistent.

### Forms (Contact / Project / Meeting)

- **C11 [P1]** All three forms render `Cancel | Save` at the bottom-right. Mobile users see this *after* scrolling 6+ fields; on a phone the buttons should sticky-bottom.
- **C12 [P1]** Validation errors only appear after submit. No inline live validation (e.g., "this email looks invalid").
- **C13 [P2]** Form drafts aren't persisted on navigation — AC asks for "saved-on-blur semantics" on Contact form (FR-CAP-5). Currently lost on accidental back-button.

### MilestoneList

- **C14 [P1]** Uses `window.confirm()` for delete (A3).
- **C15 [P1]** Block button shows an inline input for the blocker reason that pushes the row taller. Use a Popover instead.

### VoiceRecorder + QuickContactRecorder

- **C16 [P0]** No Safari fallback for MIME (A2).
- **C17 [P1]** No visual waveform / level meter. User has no feedback that audio is being captured.
- **C18 [P1]** No abort button mid-recording (only Stop, which uploads).
- **C19 [P2]** Re-record clears the previous result without confirmation.

### TouchList

- **C20 [P1]** LOW-CONFIDENCE prefix detection is regex-fragile — if the prefix format changes, badges silently disappear.
- **C21 [P3]** Each touch wraps in its own `<li>` card; for a long timeline this creates a stack of cards rather than a unified feed. Reduce the chrome.

### Grid (ColumnHeader / FilterBar / SavedViews / ExportButton)

- **C22 [P1]** ColumnHeader href uses `scroll={false}` (good), but `prefetch` isn't disabled — Next prefetches every reordered sort variant, hitting the DB extra times for nothing.
- **C23 [P1]** SavedViews uses `window.prompt()` (A3).
- **C24 [P1]** FilterBar's "Group by" reuses the same Select chrome as filter selects — no visual distinction that it's a different kind of control.
- **C25 [P2]** Saved views are per-browser; no portability across devices.
- **C26 [P3]** Export button label "Export CSV" — fine but always exports the current filter state. Some users will expect "all data."

### Kanban

- **C27 [P0]** Hover-only controls (A5).
- **C28 [P1]** No keyboard navigation across columns (arrow keys to move a card).

### IntroTree / WarmPath / ReciprocityCard / Heatmap

- **C29 [P1]** Heatmap has no legend / axis (B6).
- **C30 [P1]** ReciprocityCard balance bar direction is non-obvious (B20).
- **C31 [P2]** IntroTree depth indicator (left border) doesn't scale visually beyond ~5 levels.

### ReintroButton

- **C32 [P1]** Inline expand (B18).
- **C33 [P2]** "Copy" button has no success state — silent navigator.clipboard.writeText.
